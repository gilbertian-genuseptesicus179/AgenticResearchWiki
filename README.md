## 📘 What this is

AgenticResearchWiki is a ready-to-use Obsidian wiki and CLAUDE.md skeleton for your project.

It helps you keep project notes in one place and gives your AI tools the context they need to work with you. Use it to store project facts, goals, decisions, file maps, and notes that an agent can read fast.

This is useful if you want your docs to stay close to your code and your AI helper to know the project like you do.

You need:

- A Windows PC
- A web browser
- Obsidian installed, if you want to use the wiki view
- A file unzip tool, such as the built-in Windows Extract option
- A text editor, if you want to edit the markdown files by hand

## 📥 Download and open the files

1. Open the GitHub link above.
2. Download the repository to your PC.
3. If the file downloads as a .zip file, right-click it and choose Extract All.
4. Open the extracted folder.
5. Open the wiki in Obsidian, or open the markdown files in any text editor.

## 🧭 First-time setup

1. Put the project folder in a place you can find again, such as Documents or Desktop.
2. Open the folder and look for the main markdown files.
3. Open CLAUDE.md and add your project name, goals, and rules.
4. Fill in the wiki pages with your main project facts.
5. Keep the same folder for all future notes so your agent can follow the trail.

## 🗂️ Suggested folder use

Use the wiki to store:

- Project goals
- Main features
- Important terms
- File map
- Work notes
- Known issues
- Decisions and why you made them
- Links to related docs

## 🤖 How agents use this

AI tools work better when they can read clear project notes.

With this wiki, an agent can:

- Learn what your project does
- Find the main files faster
- Follow your rules
- Avoid repeated questions
- Keep changes closer to your project goals

## 📝 Good things to add to CLAUDE.md

Add short, plain notes like:

- What the app does
- Who uses it
- What files matter most
- What not to change
- How you want names, paths, and notes handled
- Any setup steps that matter for your project

## 🔍 How to keep it useful

- Write short notes
- Use plain words
- Keep one fact per line when you can
- Update the wiki when the project changes
- Remove old notes that no longer help
- Keep key decisions in one place

## 🧩 Works well with these tools

- Save changes often
- Keep file names clear
